关于使用postman调用api

关于office365 E5我已经再上一篇文章里提到了，在这里就不重复了（传送门）

在Azure注册应用

打开并登录Azure门户

名称随意，重定向URL：

https://app.getpostman.com/oauth2/callback

记住client_id、client_secret、tenant_id

将信息导入postman

点击Import–Import From Link导入

https://raw.githubusercontent.com/microsoftgraph/microsoftgraph-postman-collections/master/Microsoft%20Graph.postman_environment.json

选择刚加入的Microsoft Graph environment。之后点击右侧眼睛图形，然后点击Edit

输入信息

Client ID: ?
Tenant ID: ?
Client Secret: ?

“?”自行补充

接着导入(Import–Import From Link)

https://raw.githubusercontent.com/microsoftgraph/microsoftgraph-postman-collections/master/Microsoft%20Graph.postman_collection.json

会出现一个Microsoft Graph的集合

点击Edit

点击Get New Access Token，输入信息。

Token Name: ?
Callback URL: https://app.getpostman.com/oauth2/callback
Auth URL: https://login.microsoftonline.com/#tenant_id#/oauth2/v2.0/authorize
Access Token URL: https://login.microsoftonline.com/#tenant_id#/oauth2/v2.0/token
Client ID: ?
Client Secret: ?
Scope: https://graph.microsoft.com/.default

“?”自行补充

点击Request Token，登录office365 E5账号

测试

选择Microsoft Graph集合–On Behalf of a User集合–Mail集合–Get my messages–Authorization

更改type为OAuth 2.0

依次点击Get New Access Token–Request Token–登录–Use Token–Send

如果你在下方给出的信息不是

那么恭喜你，postman已经与Microsoft Graph通过api相连
